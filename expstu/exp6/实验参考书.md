
## 一、RDD 数据清洗与 DataFrame 格式化

### 1.1 实验目标

能够熟练使用 RDD 清洗数据，并规范化为 DataFrame 输入格式，掌握从原始评分数据到结构化训练数据的完整预处理流程。

### 1.2 背景知识

在实际推荐系统中，原始数据往往格式不统一、存在缺失值或异常值。MovieLens 评分数据以 `::` 作为分隔符，每行包含 `userId::movieId::rating::timestamp` 四个字段。PySpark 的 `ml.recommendation.ALS` 要求输入为 DataFrame，且列名和数据类型须与 ALS 参数匹配（如 `userId` 为整数、`rating` 为浮点数）。因此，数据预处理是推荐流水线的第一步，也是最关键的一步。

### 1.3 操作步骤

**步骤一：初始化 Spark 会话**

使用 `exputils` 工具模块快速初始化 Spark 环境：

```python
import exputils as exp
spark, sql, sc, pyspark, ml = exp.import_spark()
```

> **提示**：`exp.import_spark()` 会返回五个核心对象，其中 `spark` 是 SparkSession，`sql` 是 pyspark.sql 模块，`ml` 是 pyspark.ml 模块。后续所有实验均需先执行此初始化。

**步骤二：读取原始评分数据**

使用 `spark.read.text()` 读取文本文件，返回一个包含 `value` 列的 DataFrame，再通过 `.rdd` 转为 RDD：

- 关键 API：`spark.read.text(path)` —— 读取文本文件，每行一条记录
- 关键 API：`DataFrame.rdd` —— 将 DataFrame 转为 RDD

文件路径使用 `exp.local_file_prefix + exp.jmd('sample_movielens_ratings.txt')` 构造。

**步骤三：数据拆分与清洗**

对 RDD 中每条记录进行处理：

1. 使用 `map` 将每行按 `"::"` 拆分：`row.value.split("::")`
2. 使用 `filter` 过滤掉字段数不等于 4 的异常行：`lambda x: len(x) == 4`
3. 使用 `map` 将字段转为目标类型并构造 Row 对象：`sql.Row(userId=int(p[0]), movieId=int(p[1]), rating=float(p[2]))`

> **注意**：`timestamp` 字段（p[3]）在 ALS 训练中不需要，因此不必包含在 Row 中。

**步骤四：创建 DataFrame 并验证**

将清洗后的 RDD 转为 DataFrame，并检查数据结构：

- 关键 API：`spark.createDataFrame(rdd)` —— 将 RDD 转为 DataFrame
- 关键 API：`DataFrame.printSchema()` —— 打印表结构
- 关键 API：`DataFrame.show(truncate=False)` —— 显示前 20 行数据

> **验证要点**：确认 `userId` 和 `movieId` 为 `int` 类型，`rating` 为 `float` 类型。

### 1.4 思考题

1. 如果原始数据中存在空行或字段缺失，`filter(lambda x: len(x) == 4)` 能否完全处理？还有什么情况需要考虑？
2. 为什么要将 RDD 转为 DataFrame 而非直接使用 RDD？ALS 算法为什么不接受 RDD 输入？

## 二、调用 ALS 算法完成模型训练

### 2.1 实验目标

能够调用 ml 包算法实现 ALS 算法，完成训练过程，理解交替最小二乘法（ALS）通过矩阵分解学习用户和物品隐语义因子的基本原理。

### 2.2 背景知识

ALS（Alternating Least Squares）是协同过滤中最经典的矩阵分解算法。其核心思想是将用户-物品评分矩阵 R 分解为两个低秩矩阵的乘积：R ≈ U × V^T，其中 U 是用户隐因子矩阵，V 是物品隐因子矩阵。训练时，ALS 交替固定 U 更新 V、固定 V 更新 U，直至收敛。PySpark 的 `ml.recommendation.ALS` 是分布式实现，可处理大规模数据。

### 2.3 操作步骤

**步骤一：划分训练集和测试集**

使用 `DataFrame.randomSplit()` 将数据按 8:2 比例划分：

- 关键 API：`DataFrame.randomSplit([0.8, 0.2])` —— 返回一个列表，第一个为训练集，第二个为测试集

> **提示**：`randomSplit` 的结果具有随机性，可通过设置种子参数（如 `seed=42`）保证可复现。

**步骤二：创建 ALS 模型并设置参数**

使用 `ml.recommendation.ALS()` 创建 ALS 实例，关键参数：

| 参数 | 含义 | 推荐初始值 |
|------|------|-----------|
| `rank` | 隐因子维度 | 10 |
| `maxIter` | 最大迭代次数 | 5 |
| `regParam` | 正则化参数 | 0.01 |
| `userCol` | 用户 ID 列名 | `"userId"` |
| `itemCol` | 物品 ID 列名 | `"movieId"` |
| `ratingCol` | 评分列名 | `"rating"` |
| `coldStartStrategy` | 冷启动策略 | `"drop"` |

> **重要**：`coldStartStrategy="drop"` 会在预测时丢弃测试集中无法预测的行（即训练集未出现的用户或物品），避免评估时出现 NaN。

**步骤三：训练模型**

- 关键 API：`als.fit(training)` —— 在训练集上训练 ALS 模型，返回 `ALSModel`

**步骤四：查看模型基本信息**

训练完成后，可打印模型的用户因子和物品因子：

- 关键 API：`model.userFactors` —— 用户隐因子矩阵 DataFrame
- 关键 API：`model.itemFactors` —— 物品隐因子矩阵 DataFrame

> **提示**：`userFactors` 的 `features` 列是一个长度为 `rank` 的浮点数数组，代表该用户在隐语义空间中的向量表示。

### 2.4 思考题

1. `rank` 参数代表什么？增大或减小 `rank` 对模型有什么影响？
2. 为什么 ALS 要交替优化 U 和 V，而不是同时优化？
3. `coldStartStrategy="drop"` 与 `"nan"` 有何区别？在什么场景下应选择哪种策略？

## 三、基于用户和商品的推荐

### 3.1 实验目标

利用 ALS 模型分别实现基于用户和商品的推荐，掌握为指定用户生成 Top-N 推荐列表以及为指定商品寻找相似用户群的方法。

### 3.2 背景知识

ALS 模型训练完成后，可以利用用户因子矩阵 U 和物品因子矩阵 V 计算预测评分，进而为用户推荐其可能感兴趣的物品，或为物品找到可能感兴趣的用户。PySpark ALS 模型提供了多种推荐方法，既可以批量推荐，也可以对子集推荐。

### 3.3 操作步骤

**步骤一：为所有用户生成 Top-10 推荐**

- 关键 API：`model.recommendForAllUsers(numItems)` —— 为每个用户推荐 numItems 个物品
- 返回的 DataFrame 包含 `userId` 和 `recommendations` 两列，`recommendations` 是 `movieId` 和 `rating` 的结构体数组

使用 `.show(truncate=False)` 查看推荐结果。

**步骤二：为所有商品推荐 Top-10 用户**

- 关键 API：`model.recommendForAllItems(numUsers)` —— 为每个物品推荐 numUsers 个用户

**步骤三：为指定用户子集推荐**

1. 从 ratings 中选取若干不重复的用户：
   - 关键 API：`ratings.select(als.getUserCol()).distinct().limit(3)`
2. 对该用户子集生成推荐：
   - 关键 API：`model.recommendForUserSubset(users, 10)`

**步骤四：为指定商品子集推荐**

1. 从 ratings 中选取若干不重复的商品：
   - 关键 API：`ratings.select(als.getItemCol()).distinct().limit(3)`
2. 对该商品子集生成推荐：
   - 关键 API：`model.recommendForItemSubset(movies, 10)`

> **提示**：使用 `.sshow()` 或 `.show()` 展示结果，观察推荐的 movieId 和预测评分。

### 3.4 思考题

1. `recommendForAllUsers` 和 `recommendForUserSubset` 在计算效率上有什么差异？什么情况下应优先使用后者？
2. 推荐列表中的预测评分代表什么含义？它和用户真实评分有何区别？

## 四、FP-growth 关联规则商品推荐

### 4.1 实验目标

掌握利用 FP-growth 算法完成基于关联规则的商品推荐，理解频繁项集挖掘中支持度（Support）和置信度（Confidence）阈值的作用及调优方法。

### 4.2 背景知识

关联规则挖掘是推荐系统中的经典方法，其核心思想是：如果用户购买了商品 A，则很可能也购买商品 B。FP-growth（Frequent Pattern Growth）是一种高效的频繁项集挖掘算法，相比 Apriori 算法无需产生候选项集，通过构造 FP 树实现两次扫描即可完成挖掘。在 PySpark 中，`ml.fpm.FPGrowth` 提供了分布式实现。

### 4.3 操作步骤

**步骤一：使用小规模示例数据理解 FP-growth**

先使用 `small_fpgrowth_sample.csv` 理解 FP-growth 的基本流程：

1. 读取数据：使用 `spark.createDataFrame()` 手动构造一个包含 `id` 和 `items` 列的 DataFrame，其中 `items` 列是整数数组类型
2. 示例数据结构：
   ```
   id | items
   0  | [1, 2, 5]
   1  | [1, 2, 3, 5]
   2  | [1, 2]
   ...
   ```

> **提示**：可以直接用 Python 列表构造数据，不需要从文件读取。

**步骤二：创建 FP-growth 模型并设置参数**

- 关键 API：`ml.fpm.FPGrowth(itemsCol="items", minSupport=0.5, minConfidence=0.6)`
- 参数说明：
  - `itemsCol`：事务项集所在列名
  - `minSupport`：最小支持度阈值（0~1），低于此阈值的项集不被视为频繁项集
  - `minConfidence`：最小置信度阈值（0~1），低于此阈值的规则被过滤

**步骤三：训练模型并查看结果**

- 关键 API：`fpGrowth.fit(df)` —— 训练 FP-growth 模型
- 关键 API：`model.freqItemsets.show()` —— 查看频繁项集及其频次
- 关键 API：`model.associationRules.show()` —— 查看关联规则（含支持度、置信度、提升度）
- 关键 API：`model.transform(df).show()` —— 对原始数据进行预测，为每条事务补充推荐项

**步骤四：使用完整交易数据 `user_transactions.csv` 进行关联规则挖掘**

1. 读取 `user_transactions.csv`，该文件包含 `userId` 和 `items` 两列，`items` 列是以空格分隔的电影 ID 字符串
2. 需要将 `items` 字符串拆分为整数数组：使用 `pyspark.sql.functions.split()` 和 `pyspark.sql.functions.expr()` 或 UDF
3. 创建 FP-growth 模型，尝试不同的 `minSupport` 和 `minConfidence` 组合，观察规则数量的变化

> **提示**：将字符串列转为数组列可使用 `F.split("items", " ").cast("array<int>")` 或先定义 UDF。

### 4.4 思考题

1. `minSupport` 从 0.5 降低到 0.1，频繁项集的数量如何变化？计算开销如何变化？
2. `lift`（提升度）大于 1 说明什么？如果某条规则的置信度很高但提升度接近 1，这条规则有用吗？
3. FP-growth 与 Apriori 的本质区别是什么？为什么 FP-growth 通常更高效？

## 五、基于评分次数的电影推荐

### 5.1 实验目标

能够实现基于评分次数的电影推荐，理解通过用户活跃度（评分频次）筛选候选邻居集以提升推荐效率的思路。

### 5.2 背景知识

在实际推荐场景中，用户的评分次数差异巨大——有些用户评分成千上万次（活跃用户），有些仅评分一两次（冷启动用户）。评分次数可以作为用户活跃度的度量，用于：1）筛选高活跃用户作为"邻居"，提高协同过滤的效率；2）识别冷启动用户，采用特殊推荐策略。此外，电影被评分的次数也反映了其流行度，可以作为推荐的重要参考维度。

### 5.3 操作步骤

**步骤一：统计用户评分次数**

从 ratings DataFrame 中统计每个用户的评分次数：

- 关键 API：`ratings.groupBy('userId').count()` —— 按 userId 分组计数
- 关键 API：`.orderBy('count', ascending=False)` —— 按评分次数降序排列
- 使用 `.show(10, False)` 查看评分次数最多的前 10 名用户

**步骤二：统计电影评分次数**

类似地，统计每部电影的评分次数：

- 关键 API：`ratings.groupBy('movieId').count()`
- 按 `count` 降序排列，查看最受欢迎的电影

**步骤三：筛选高活跃用户**

根据评分次数筛选活跃用户（如评分次数 >= 某阈值），作为 ALS 训练的候选邻居集：

- 关键 API：`DataFrame.filter()` 或 `DataFrame.where()` —— 按条件过滤
- 示例逻辑：`active_users = user_counts.filter(user_counts['count'] >= 30)`

**步骤四：读取 `user_activity.csv` 验证**

读取预生成的 `user_activity.csv` 文件，验证用户活跃度统计结果：

- 文件格式：`userId, rating_count, avg_rating`
- 使用 `spark.read.csv(path, header=True, inferSchema=True)` 读取
- 对比自己计算的结果是否一致

2. 如果仅使用高活跃用户的评分数据训练 ALS 模型，模型性能会提升还是下降？为什么？

## 六、混合推荐规则与已观看项目过滤

### 6.1 实验目标

组建混合推荐规则，并且过滤用户已观看过的项目，掌握推荐结果去重与业务规则过滤的实现方法。

### 6.2 背景知识

单一推荐算法往往有局限性——ALS 对新用户/新物品效果差（冷启动），FP-growth 无法生成个性化评分预测。混合推荐（Hybrid Recommendation）将多种推荐结果融合，取长补短。此外，推荐系统中一个重要的业务规则是：不要向用户推荐其已经观看/购买过的项目。因此，过滤已观看项目是推荐后处理的必要环节。

### 6.3 操作步骤

**步骤一：获取 ALS 推荐结果**

从前面实验中获取 `model.recommendForAllUsers(10)` 的结果，并将嵌套的 `recommendations` 数组展开：

- 关键 API：`F.explode("recommendations")` —— 将数组列展开为多行
- 展开后可提取 `movieId` 和 `prediction`（预测评分）

**步骤二：获取 FP-growth 关联推荐结果**

对 `user_transactions` 数据使用 FP-growth 模型的 `transform()` 方法，获取基于关联规则的推荐结果。

**步骤三：合并两种推荐**

使用 `DataFrame.union()` 或 `unionByName()` 将 ALS 推荐和 FP-growth 推荐合并：

- 关键 API：`als_recs.unionByName(fp_recs)` —— 按列名合并两个 DataFrame

**步骤四：过滤已观看项目**

1. 获取每个用户已观看的电影列表：
   - 关键 API：`ratings.groupBy("userId").agg(F.collect_set("movieId").alias("watched"))`
2. 将推荐结果与已观看列表进行左连接
3. 过滤掉推荐列表中已观看的电影：
   - 关键 API：`F.array_contains("watched", "movieId")` —— 判断电影是否在已观看列表中
   - 取反：`~F.array_contains(...)`

**步骤五：推荐结果去重**

同一部电影可能同时被 ALS 和 FP-growth 推荐，需要对 `(userId, movieId)` 去重：

- 关键 API：`DataFrame.dropDuplicates(["userId", "movieId"])` —— 按指定列去重

### 6.4 思考题

1. 混合推荐中，ALS 推荐和 FP-growth 推荐的权重应如何分配？有什么策略？
2. 如果某个用户在 FP-growth 中没有关联规则命中，混合推荐会退化为哪种方式？如何保证推荐覆盖率？

## 七、使用 RegressionEvaluator 评估 ALS 模型

### 7.1 实验目标

能够使用 RegressionEvaluator 对 ALS 模型进行评估，掌握 RMSE（均方根误差）和 MAE（平均绝对误差）等回归评估指标的计算与解读方法。

### 7.2 背景知识

模型评估是推荐系统开发中不可忽视的环节。对于评分预测任务，常用的评估指标包括：

- **RMSE（Root Mean Square Error）**：均方根误差，对大误差更敏感，公式为 √(Σ(ŷ - y)² / n)
- **MAE（Mean Absolute Error）**：平均绝对误差，对异常值更鲁棒，公式为 Σ|ŷ - y| / n

PySpark 提供 `ml.evaluation.RegressionEvaluator` 来计算这些指标。

### 7.3 操作步骤

**步骤一：生成预测结果**

使用训练好的 ALS 模型对测试集进行预测：

- 关键 API：`model.transform(test)` —— 对测试集生成预测评分
- 查看预测结果的 `prediction` 列

**步骤二：计算 RMSE**

创建 RegressionEvaluator 实例并计算 RMSE：

- 关键 API：`ml.evaluation.RegressionEvaluator(metricName="rmse", labelCol="rating", predictionCol="prediction")`
- 关键 API：`evaluator.evaluate(predictions)` —— 计算评估指标值

**步骤三：计算 MAE**

修改 `metricName` 为 `"mae"` 重新评估：

- 关键 API：`ml.evaluation.RegressionEvaluator(metricName="mae", labelCol="rating", predictionCol="prediction")`

**步骤四：分析评估结果**

- RMSE 和 MAE 的值越接近 0 说明模型预测越准确
- 对于 1-5 分的评分体系，RMSE 在 0.8-1.2 范围内属于可接受水平
- 如果 RMSE 远大于 MAE，说明存在少量较大的预测偏差

### 7.4 思考题

1. RMSE 和 MAE 哪个对异常值更敏感？在实际推荐系统中，应优先关注哪个指标？
2. 如果 RMSE 为 1.5，在 1-5 分制下意味着什么？模型是否可用？

## 八、稀疏性问题与冷启动应对

### 8.1 实验目标

理解协同过滤中用户-物品评分矩阵的稀疏性问题，掌握 ALS 算法处理大规模稀疏矩阵的技术优势及冷启动（Cold Start）场景的基本应对思路。

### 8.2 背景知识

协同过滤的核心困境是数据稀疏性——在百万级用户和物品的系统中，用户通常只评价了极少部分物品，评分矩阵的填充率可能不到 1%。ALS 通过矩阵分解将稀疏矩阵压缩为两个低秩矩阵，天然具备处理稀疏数据的能力。然而，对于全新的用户或物品（即训练集中未出现的 ID），ALS 无法给出预测——这就是"冷启动"问题。

### 8.3 操作步骤

**步骤一：分析评分矩阵的稀疏度**

1. 统计用户数量、电影数量、评分总量
2. 计算评分矩阵的稀疏度 = 1 - (评分总量 / (用户数 × 电影数))

- 关键 API：`DataFrame.count()` —— 计算行数
- 关键 API：`DataFrame.select(F.countDistinct("userId"))` —— 计算不重复用户数

**步骤二：读取冷启动数据**

读取 `cold_start_users.csv`，该文件包含 10 个只有 1-3 条评分记录的新用户：

- 关键 API：`spark.read.csv("cold_start_users.csv", header=True, inferSchema=True)`

**步骤三：观察冷启动问题**

1. 使用原始训练好的 ALS 模型对冷启动用户进行预测
2. 观察哪些冷启动用户无法获得预测结果（NaN）
3. 尝试将冷启动用户的评分加入训练集重新训练，观察预测效果

**步骤四：冷启动应对策略**

## 九、交叉验证与超参数优化

### 9.1 实验目标

掌握训练集与测试集的划分方法，能够对 ALS 模型进行交叉验证，通过调整隐因子维度（rank）、正则化参数（regParam）和迭代次数（maxIter）等超参数优化模型性能。

### 9.2 背景知识

ALS 模型的性能高度依赖超参数设置。`rank` 决定了隐因子空间的维度——过低会欠拟合，过高会过拟合且计算成本大；`regParam` 控制正则化强度，防止过拟合；`maxIter` 控制迭代次数，过少则未收敛，过多则浪费时间。通过交叉验证（Cross-Validation）系统地搜索最优参数组合，是提升模型性能的标准做法。

### 9.3 操作步骤

**步骤一：构建参数网格**

使用 `ml.tuning.ParamGridBuilder` 构建超参数搜索空间：

- 关键 API：`ml.tuning.ParamGridBuilder()`
- 参数组合建议：
  - `rank`: [5, 10, 20]
  - `regParam`: [0.01, 0.1, 0.5]
  - `maxIter`: [5, 10]

示例：
```python
paramGrid = ml.tuning.ParamGridBuilder() \
    .addGrid(als.rank, [5, 10, 20]) \
    .addGrid(als.regParam, [0.01, 0.1, 0.5]) \
    .addGrid(als.maxIter, [5, 10]) \
    .build()
```

**步骤二：创建交叉验证器**

- 关键 API：`ml.tuning.CrossValidator(estimator=als, estimatorParamMaps=paramGrid, evaluator=evaluator, numFolds=3)`
- 参数说明：
  - `estimator`：待评估的模型（ALS 实例）
  - `estimatorParamMaps`：参数网格
  - `evaluator`：评估指标（使用前面创建的 RegressionEvaluator）
  - `numFolds`：折数（3 折或 5 折）

**步骤三：运行交叉验证**

- 关键 API：`cvModel.fit(training)` —— 执行交叉验证
- 此过程可能较慢，因为需要训练 `len(paramGrid) × numFolds` 个模型

**步骤四：查看最优参数和模型**

- 关键 API：`cvModel.bestModel` —— 获取最优模型
- 关键 API：`cvModel.bestModel.rank` —— 查看最优的 rank 值
- 关键 API：`cvModel.avgMetrics` —— 查看每组参数的平均评估指标

**步骤五：用最优模型评估测试集**

- 使用 `cvModel.bestModel.transform(test)` 生成预测
- 用 RegressionEvaluator 计算测试集 RMSE，与手动调参的结果对比

## 十、隐因子矩阵的可解释性分析与可视化

### 10.1 实验目标

能够利用 ALS 模型提取的用户隐因子矩阵（userFactors）和物品隐因子矩阵（itemFactors），进行推荐结果的可解释性分析与可视化展示。

### 10.2 背景知识

ALS 的矩阵分解将每个用户和每部电影映射到一个 `rank` 维的隐语义空间中。虽然这些维度没有直接的语义解释，但通过对隐因子向量的分析，可以发现潜在的语义结构——例如某些维度可能对应"动作 vs 文艺"或"老片 vs 新片"的偏好。将高维隐因子降维到二维空间进行可视化，可以直观地观察用户和物品的分布关系。

### 10.3 操作步骤

**步骤一：提取隐因子矩阵**

- 关键 API：`model.userFactors` —— 获取用户隐因子矩阵（列：`id`, `features`）
- 关键 API：`model.itemFactors` —— 获取物品隐因子矩阵

将 `features` 列（数组类型）转为 Python 列表，便于后续处理：

- 关键 API：`DataFrame.collect()` —— 将 DataFrame 收集到 Driver 端

> **注意**：`collect()` 会将全量数据拉到 Driver 内存，仅适用于小规模数据。实验中的 MovieLens 子集数据量小，可以安全使用。

**步骤二：将隐因子转为 NumPy 矩阵**

```python
import numpy as np
# user_factors: List[Row]，每行包含 id 和 features 数组
user_vectors = np.array([row.features for row in user_factors])
item_vectors = np.array([row.features for row in item_factors])
```

**步骤三：使用 PCA 降维到二维**

使用 `sklearn.decomposition.PCA` 将高维隐因子降维到二维：

- 关键 API：`PCA(n_components=2).fit_transform(vectors)`

**步骤四：绘制散点图**

使用 `matplotlib` 绘制用户和物品在二维空间中的分布：

- 关键 API：`plt.scatter()` —— 绘制散点图
- 建议用不同颜色区分用户和物品
- 可以标注部分电影名称（关联 `movies.csv` 中的 title 列）

**步骤五：分析隐因子的语义**

1. 观察散点图中是否有明显的聚类
2. 尝试找出某些维度对应的潜在语义（如按电影类型着色观察分布）
3. 选择一个用户，计算其隐因子向量与所有电影隐因子向量的点积，验证 Top-N 推荐结果是否与 ALS 模型的推荐一致

> **提示**：点积计算可使用 `np.dot(user_vec, item_vec.T)`，结果越大说明用户对该物品的预测评分越高。
