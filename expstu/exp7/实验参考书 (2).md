
## 一、Criteo数据集预处理与特征工程

### 1.1 实验目标

掌握CTR预估任务中原始数据的预处理流程，能够对Criteo广告点击率数据集进行特征工程处理，包括连续特征的截断与对数变换、类别特征的词表构建与编码映射，以及特征尺寸文件的生成与解读。

### 1.2 背景知识

Criteo Display Advertising Challenge数据集是CTR预估领域最经典的基准数据集之一，来源于Criteo实验室的真实广告展示日志。该数据集的目标是根据用户特征和广告特征预测某条广告是否会被用户点击，属于二分类问题。原始数据集包含约4500万条训练样本和600万条测试样本，每条样本由一个标签（0或1）、13个连续特征（整数类型）和26个类别特征（匿名哈希值）组成。连续特征代表用户或广告的数值型属性（如用户活跃度、广告展示次数等），类别特征则代表离散型属性（如广告位ID、用户设备类型等），均已经过匿名化处理以保护隐私。

在实际CTR预估任务中，原始数据往往存在以下问题：连续特征的数值范围差异巨大，可能从0到数十万不等，直接输入模型会导致梯度不稳定；类别特征的取值空间极为庞大（某些字段的唯一值可达数百万），且存在大量低频取值，需要进行词表截断；数据中存在大量缺失值，需要合理的填充策略。因此，数据预处理是CTR预估流水线的第一步，也是影响模型最终性能的关键环节。

### 1.3 操作步骤

**步骤一：运行预处理脚本**

使用`utils/dataPreprocess.py`对原始Criteo数据进行预处理：

```python
python utils/dataPreprocess.py
```

该脚本会读取`data/raw/`目录下的原始数据文件，经过处理后生成`data/train.txt`、`data/test.txt`和`data/feature_sizes.txt`。

- 关键类：`ContinuousFeatureGenerator` —— 连续特征处理器，负责截断和填充
- 关键类：`CategoryDictGenerator` —— 类别特征处理器，负责词表构建和编码映射

**步骤二：理解连续特征处理逻辑**

`ContinuousFeatureGenerator`类对13个连续特征进行截断处理。截断阈值基于各特征的95%分位数设定：

```python
continous_clip = [20, 600, 100, 50, 64000, 500, 100, 50, 500, 10, 10, 10, 50]
```

- 超过截断值的特征值被裁剪到截断值，去除极端异常值的影响
- 缺失值统一填充为0.0，保证输入的完整性
- `gen(idx, val)`方法：如果值为空字符串则返回0.0，否则返回浮点数值

> **注意**：截断阈值的设定对模型性能有直接影响。过于激进的截断会丢失有效信息，过于宽松则无法有效抑制异常值。95%分位数是常用的折中策略。

**步骤三：理解类别特征处理逻辑**

`CategoryDictGenerator`类对26个类别特征构建词表，核心流程如下：

1. 遍历训练数据，统计每个类别值的出现频次
2. 按频次降序排列，过滤掉出现次数低于`cutoff`阈值（默认为10）的低频类别
3. 为保留的类别值分配从1开始的整数索引，索引0保留给未登录词（`<unk>`）
4. `gen(idx, key)`方法：如果类别值不在词表中，返回0（`<unk>`索引），否则返回对应索引

> **提示**：低频过滤的cutoff参数需要根据数据规模调整。cutoff越大，词表越小、特征空间越紧凑，但可能丢失有用的长尾信息；cutoff越小，词表越大、信息保留越多，但会增加Embedding层的参数量。

**步骤四：生成feature_sizes.txt**

预处理脚本的最后一步是生成`feature_sizes.txt`文件：

```python
sizes = [1] * len(continous_features) + dict_sizes
```

- 连续特征的size均为1（表示单一浮点值，使用1维Embedding）
- 类别特征的size为各自词表的长度
- 该文件在模型初始化时被读取，用于确定各字段的Embedding层输入维度

**步骤五：在CriteoDataset中理解数据加载**

`data/dataset.py`中的`CriteoDataset`类将预处理后的数据转换为模型可用的张量格式：

- `Xi`（特征索引）：连续特征的索引为0，类别特征的索引为词表映射后的整数
- `Xv`（特征值）：连续特征经过`log1p(max(x, 0))`变换，类别特征值为1.0（独热编码激活标记）
- `__getitem__`返回三元组`(Xi, Xv, target)`用于训练，`(Xi, Xv)`用于测试

> **关键理解**：log1p变换（即log(1+x)）是一种常用的连续特征归一化方法，能有效压缩数值范围，使梯度更新更稳定。对负值先取max(x, 0)确保输入非负。

### 1.4 思考题

1. 如果将cutoff阈值从10提高到100，词表大小会如何变化？模型参数量如何变化？对模型性能有何影响？
2. 为什么要对连续特征做log1p变换而不是直接标准化（Z-Score）？在什么情况下log1p优于Z-Score？
3. 类别特征中未登录词（`<unk>`）的索引为0，这意味着所有未见过的类别值共享同一个Embedding向量，这种设计的优缺点是什么？

## 二、DeepFM模型架构设计与实现

### 2.1 实验目标

理解DeepFM模型的整体架构设计，掌握FM组件与DNN组件共享特征嵌入的机制，能够使用PyTorch框架实现DeepFM模型类的定义与初始化。

### 2.2 背景知识

DeepFM由Huifeng Guo等人在2017年的论文《DeepFM: A Factorization-Machine based Neural Network for CTR Prediction》中提出。该模型的核心创新在于将因子分解机（FM）和深度神经网络（DNN）融合为一个端到端的学习框架，FM组件负责捕捉低阶特征交互（一阶和二阶），DNN组件负责捕捉高阶特征交互，两者共享同一套特征嵌入参数。

DeepFM的设计哲学源于对CTR预估中两个关键能力的兼顾：记忆能力（Memorization）和泛化能力（Generalization）。FM组件通过显式的二阶特征交叉实现记忆能力，能够记住"当特征A和特征B同时出现时，点击率显著提升"这类规则；DNN组件通过多层非线性变换实现泛化能力，能够发现FM无法捕捉的复杂高阶交互模式。与Wide&Deep模型相比，DeepFM的最大优势在于无需手工设计交叉特征——FM组件自动学习二阶交互，大大降低了特征工程的复杂度。

在PyTorch实现中，DeepFM继承自`nn.Module`，通过`__init__`方法定义所有可学习参数，通过`forward`方法实现前向传播逻辑。共享嵌入机制确保FM和DNN从相同的特征表示中学习，促进低阶和高阶交互的互补增强。

### 2.3 操作步骤

**步骤一：阅读模型类定义**

打开`model/DeepFM.py`，阅读`DeepFM`类的`__init__`方法，理解各组件的初始化逻辑：

```python
class DeepFM(nn.Module):
    def __init__(self, feature_sizes, embedding_size=4,
                 hidden_dims=[32, 32], num_classes=1, dropout=[0.5, 0.5],
                 use_cuda=True, verbose=False):
```

关键参数说明：

| 参数 | 含义 | 默认值 |
|------|------|--------|
| `feature_sizes` | 各字段的特征维度列表 | 必填 |
| `embedding_size` | 嵌入向量维度 | 4 |
| `hidden_dims` | DNN隐藏层维度列表 | [32, 32] |
| `num_classes` | 输出类别数 | 1 |
| `dropout` | 各隐藏层Dropout率 | [0.5, 0.5] |
| `use_cuda` | 是否使用GPU | True |

**步骤二：理解FM一阶嵌入层**

FM一阶部分为每个字段创建一个Embedding层，输出维度为1：

```python
self.fm_first_order_embeddings = nn.ModuleList(
    [nn.Embedding(feature_size, 1) for feature_size in self.feature_sizes])
```

- 每个字段有独立的Embedding权重矩阵，形状为`(feature_size, 1)`
- 一阶嵌入的输出即为该字段的线性权重，类似逻辑回归中的特征权重

**步骤三：理解FM二阶嵌入层（与DNN共享）**

FM二阶部分为每个字段创建一个Embedding层，输出维度为`embedding_size`：

```python
self.fm_second_order_embeddings = nn.ModuleList(
    [nn.Embedding(feature_size, self.embedding_size) for feature_size in self.feature_sizes])
```

- 每个字段的二阶嵌入矩阵形状为`(feature_size, embedding_size)`
- **这是FM组件和DNN组件共享的嵌入层**，是DeepFM的核心设计

**步骤四：理解DNN隐藏层初始化**

DNN隐藏层通过循环动态创建：

```python
all_dims = [self.field_size * self.embedding_size] + self.hidden_dims + [self.num_classes]
for i in range(1, len(hidden_dims) + 1):
    setattr(self, 'linear_' + str(i), nn.Linear(all_dims[i-1], all_dims[i]))
    setattr(self, 'batchNorm_' + str(i), nn.BatchNorm1d(all_dims[i]))
    setattr(self, 'dropout_' + str(i), nn.Dropout(dropout[i-1]))
```

- DNN输入维度为`field_size * embedding_size`（即所有字段嵌入向量的拼接长度）
- 每层包含三个子模块：全连接层（Linear）、批归一化层（BatchNorm）、Dropout层
- `setattr`动态命名使得层结构可扩展，支持任意深度的DNN配置

> **提示**：尝试修改`hidden_dims`参数（如改为[64, 64, 32]），观察模型参数量的变化和训练效果的差异。

2. 共享嵌入机制有什么优缺点？如果FM和DNN使用独立的嵌入层，模型效果可能如何变化？
3. `setattr`动态创建层的方式与直接在`__init__`中声明固定层相比，有什么优势和潜在风险？

## 三、FM组件的前向传播与特征交互

### 3.1 实验目标

深入理解FM组件的前向传播过程，掌握一阶特征交互和二阶特征交叉的数学推导与代码实现，理解FM化简公式的计算效率优势。

### 3.2 背景知识

因子分解机（Factorization Machine, FM）由Steffen Rendle于2010年提出，是推荐系统中最经典的特征交互建模方法之一。FM的核心思想是为每个特征学习一个低维隐向量，通过隐向量间的内积来建模特征间的交互关系，从而将特征交叉的参数量从O(n^2)降低到O(kn)，其中k是隐向量维度，n是特征数量。

FM的预测公式为：y = w_0 + Σw_i·x_i + ΣΣ(v_i·v_j)·x_i·x_j，其中第一项是全局偏置，第二项是一阶线性项，第三项是二阶交互项。直接计算二阶项的复杂度为O(k·n^2)，因为需要遍历所有特征对。但通过FM的化简公式，可以将计算复杂度降低到O(k·n)：ΣΣ(v_i·v_j)·x_i·x_j = 0.5·[(Σv_i·x_i)^2 - Σ(v_i·x_i)^2]。这个化简的关键在于将"两两交叉"转化为"和的平方减去平方的和"，避免了显式地遍历所有特征对。

在DeepFM中，FM组件的一阶部分负责捕捉各特征的独立贡献（线性效应），二阶部分负责捕捉特征间的两两交互效应。这两部分的信息与DNN组件的高阶交互信息共同构成最终的预测结果。

### 3.3 操作步骤

**步骤一：理解一阶FM计算**

在`forward`方法中，一阶FM的计算如下：

```python
fm_first_order_emb_arr = [(torch.sum(emb(Xi[:, i, :]), 1).t() * Xv[:, i]).t()
                          for i, emb in enumerate(self.fm_first_order_embeddings)]
fm_first_order = torch.cat(fm_first_order_emb_arr, 1)
```

- 对每个字段i，将特征索引`Xi[:,i,:]`通过对应的Embedding层
- `torch.sum(emb(...), 1)`对嵌入向量在最后维度求和（一阶嵌入输出为1维，求和后为标量）
- `.t() * Xv[:,i]`将嵌入值与特征值相乘（转置是为了广播对齐）
- 最终拼接所有字段的结果，得到形状为`(batch_size, field_size)`的一阶输出

**步骤二：理解二阶FM计算（化简公式）**

二阶FM的计算分为三步：

第一步：计算各字段的嵌入值与特征值的乘积：

```python
fm_second_order_emb_arr = [(torch.sum(emb(Xi[:, i, :]), 1).t() * Xv[:, i]).t()
                           for i, emb in enumerate(self.fm_second_order_embeddings)]
```

第二步：计算"和的平方"——先求和再平方：

```python
fm_sum_second_order_emb = sum(fm_second_order_emb_arr)
fm_sum_second_order_emb_square = fm_sum_second_order_emb * fm_sum_second_order_emb  # (x+y)^2
```

第三步：计算"平方的和"——先平方再求和：

```python
fm_second_order_emb_square = [item*item for item in fm_second_order_emb_arr]
fm_second_order_emb_square_sum = sum(fm_second_order_emb_square)  # x^2+y^2
```

最终，二阶FM输出为两者之差的一半：

```python
fm_second_order = (fm_sum_second_order_emb_square - fm_second_order_emb_square_sum) * 0.5
```

> **关键理解**：这个化简公式的直觉是——"所有特征对的内积之和"等于"所有特征嵌入之和的平方"减去"各特征嵌入平方之和"，再除以2（因为交叉项被计算了两次）。

**步骤三：理解二阶嵌入的共享机制**

FM二阶嵌入的中间结果被传递给DNN组件：

```python
deep_emb = torch.cat(fm_second_order_emb_arr, 1)
```

这行代码将各字段的二阶嵌入向量（经过特征值加权后）拼接为一个长向量，作为DNN的输入。**这就是FM和DNN共享嵌入的具体体现**——同一组嵌入参数既参与了FM的二阶交互计算，又被输入到DNN中进行高阶交互建模。

2. 二阶FM输出`fm_second_order`的形状是什么？为什么最后通过`torch.sum(..., 1)`求和才能得到标量预测值？
3. 化简公式中的0.5系数来自哪里？如果去掉0.5会对模型训练产生什么影响？

## 四、DNN组件的前向传播与高阶特征学习

### 4.1 实验目标

掌握DeepFM中DNN组件的前向传播过程，理解多层全连接网络如何从共享嵌入中学习高阶特征交互，以及BatchNorm和Dropout在训练中的作用。

### 4.2 背景知识

深度神经网络（DNN）通过多层非线性变换自动学习特征间的高阶交互关系。在DeepFM中，DNN的输入是FM二阶嵌入的拼接向量，经过多层全连接层、批归一化层和Dropout层的组合处理，最终输出一个高阶特征交互的表示。与FM组件显式建模二阶交互不同，DNN组件隐式地学习任意阶次的特征交互——理论上，足够深度的DNN可以逼近任意复杂度的交互函数。

BatchNorm（批归一化）是2015年由Ioffe和Szegedy提出的技术，通过对每个mini-batch的激活值进行归一化，加速训练收敛并起到一定的正则化效果。Dropout是2014年由Srivastava等人提出的正则化技术，在训练时随机丢弃一部分神经元，防止模型过度依赖某些特定特征，从而提升泛化能力。在DeepFM的DNN组件中，每个隐藏层都同时使用了BatchNorm和Dropout，这是一种常见的深度学习工程实践。

### 4.3 操作步骤

**步骤一：理解DNN输入构造**

DNN的输入来自FM二阶嵌入的拼接：

```python
deep_emb = torch.cat(fm_second_order_emb_arr, 1)
deep_out = deep_emb
```

- `fm_second_order_emb_arr`是各字段嵌入向量的列表
- `torch.cat(..., 1)`在第1维（特征维）上拼接，得到形状为`(batch_size, field_size * embedding_size)`的输入张量
- 例如：39个字段 × 4维嵌入 = 156维输入

**步骤二：理解多层前向传播**

DNN通过循环逐层前向传播：

```python
for i in range(1, len(self.hidden_dims) + 1):
    deep_out = getattr(self, 'linear_' + str(i))(deep_out)
    deep_out = getattr(self, 'batchNorm_' + str(i))(deep_out)
    deep_out = getattr(self, 'dropout_' + str(i))(deep_out)
```

每层的计算流程为：
1. `Linear`：线性变换，y = Wx + b
2. `BatchNorm1d`：批归一化，对激活值进行标准化
3. `Dropout`：随机丢弃神经元（仅训练时生效）

> **注意**：原始实现中DNN隐藏层之间没有使用激活函数（如ReLU），这在一定程度上限制了模型的非线性表达能力。建议在实验中尝试在BatchNorm之后添加ReLU激活，观察对模型性能的影响。

**步骤三：理解DNN输出维度**

DNN的最终输出维度为`num_classes`（默认为1），对应二分类任务的Logit值：

```python
all_dims = [self.field_size * self.embedding_size] + self.hidden_dims + [self.num_classes]
```

- 对于CTR预估任务，`num_classes=1`，输出为单个标量Logit
- 该Logit值需要通过Sigmoid函数转换为点击概率

### 4.4 思考题

1. 为什么DNN的输入是FM二阶嵌入的拼接而不是一阶嵌入？如果使用一阶嵌入作为DNN输入会怎样？
2. 原始实现中DNN隐藏层之间缺少激活函数，这对模型的表达能力有什么影响？尝试添加ReLU后观察训练效果的变化。
3. BatchNorm和Dropout的顺序是否重要？如果交换两者的顺序（先Dropout再BatchNorm），训练效果会怎样？

## 五、模型输出组合与损失函数

### 5.1 实验目标

掌握DeepFM最终输出的组合方式，理解FM组件和DNN组件的输出如何融合，以及二元交叉熵损失函数的选择依据与计算方法。

### 5.2 背景知识

DeepFM的最终预测值是FM组件和DNN组件的加权和。这种加法组合方式简单有效，保证了低阶交互和高阶交互的信息都能被充分利用。最终输出的Logit值需要通过Sigmoid函数转换为概率值后才能用于CTR预测。在训练时，使用二元交叉熵损失（Binary Cross Entropy, BCE）作为优化目标，BCE是二分类任务的标准损失函数，能够有效地衡量预测概率与真实标签之间的差距。

BCE的公式为：L = -[y·log(σ(z)) + (1-y)·log(1-σ(z))]，其中y是真实标签（0或1），z是模型输出的Logit值，σ是Sigmoid函数。PyTorch提供的`binary_cross_entropy_with_logits`函数内部集成了Sigmoid操作，直接接受Logit值作为输入，避免了手动计算Sigmoid可能带来的数值不稳定性问题（如Logit值过大或过小时Sigmoid的梯度消失）。

### 5.3 操作步骤

**步骤一：理解输出组合逻辑**

在`forward`方法中，最终输出为三个部分的求和加上偏置项：

```python
total_sum = torch.sum(fm_first_order, 1) + \
            torch.sum(fm_second_order, 1) + torch.sum(deep_out, 1) + self.bias
```

- `torch.sum(fm_first_order, 1)`：一阶FM输出在字段维度上求和，得到标量
- `torch.sum(fm_second_order, 1)`：二阶FM输出在嵌入维度上求和，得到标量
- `torch.sum(deep_out, 1)`：DNN输出在特征维度上求和，得到标量
- `self.bias`：全局偏置项，初始化为随机值

> **提示**：这种加法组合意味着三个组件对最终预测的贡献是等权的。如果希望控制各组件的影响，可以考虑引入可学习的权重系数。

**步骤二：理解训练损失函数**

在`fit`方法中，使用二元交叉熵损失：

```python
criterion = F.binary_cross_entropy_with_logits
# ...
total = model(xi, xv)
loss = criterion(total, y)
```

- `binary_cross_entropy_with_logits`接受Logit值和真实标签作为输入
- 内部先对Logit值应用Sigmoid，然后计算BCE损失
- 对于CTR预估中常见的正负样本不平衡问题，可以考虑设置`pos_weight`参数进行类别加权

**步骤三：理解评估指标**

在`check_accuracy`方法中，使用准确率作为评估指标：

```python
preds = (torch.sigmoid(total) > 0.5)
num_correct += (preds == y).sum()
acc = float(num_correct) / num_samples
```

- 先通过Sigmoid将Logit值转换为概率，再以0.5为阈值进行二分类判断
- 计算预测正确的样本数占总样本数的比例

> **重要**：准确率虽然直观，但在CTR预估中正负样本通常极不平衡（点击率往往低于5%），此时准确率可能产生误导。建议在实验中增加AUC（Area Under ROC Curve）和Log Loss作为辅助评估指标。

## 六、模型训练流程与优化策略

### 6.1 实验目标

掌握DeepFM模型的完整训练流程，理解数据加载、训练循环、验证评估的实现方法，以及Adam优化器的配置策略。

### 6.2 背景知识

深度学习模型的训练是一个迭代优化的过程，通过反向传播算法计算损失函数对模型参数的梯度，再由优化器根据梯度更新参数。在CTR预估任务中，由于数据量通常很大（百万到亿级样本），使用Mini-batch随机梯度下降是标准做法——每次迭代仅处理一小批样本，既保证了梯度估计的合理性，又控制了内存开销。

Adam（Adaptive Moment Estimation）优化器是目前最常用的深度学习优化器之一，由Kingma和Ba于2015年提出。Adam为每个参数自适应地调整学习率，利用梯度的一阶矩估计（均值）和二阶矩估计（方差）来动态调整更新步长。在CTR预估任务中，特征嵌入的梯度通常非常稀疏（每个batch只有少量特征被激活），Adam的自适应学习率机制对稀疏梯度场景特别有效。

### 6.3 操作步骤

**步骤一：理解数据加载与划分**

在`main.py`中，数据加载和划分的代码如下：

```python
train_data = CriteoDataset('./data', train=True)
dataset_size = len(train_data)
num_train = int(dataset_size * 0.8)

indices = list(range(dataset_size))
train_indices = indices[:num_train]
val_indices = indices[num_train:]

loader_train = DataLoader(train_data, batch_size=100,
                          sampler=sampler.SubsetRandomSampler(train_indices))
loader_val = DataLoader(train_data, batch_size=100,
                        sampler=sampler.SubsetRandomSampler(val_indices))
```

- 加载训练数据后，按80:20比例随机划分为训练集和验证集
- 使用`SubsetRandomSampler`确保训练时数据顺序随机化
- 批量大小为100，可根据GPU内存调整

> **注意**：当前划分方式是简单地按索引顺序切分，而非随机打乱后切分。如果数据本身有某种排列顺序（如按时间排序），可能导致训练集和验证集的分布不一致。建议在实际应用中使用`random.shuffle(indices)`后再划分。

**步骤二：理解模型训练循环**

在`fit`方法中，训练循环的实现如下：

```python
model = self.train().to(device=self.device)
criterion = F.binary_cross_entropy_with_logits

for _ in range(epochs):
    for t, (xi, xv, y) in enumerate(loader_train):
        xi = xi.to(device=self.device, dtype=self.dtype)
        xv = xv.to(device=self.device, dtype=torch.float)
        y = y.to(device=self.device, dtype=torch.float)

        total = model(xi, xv)
        loss = criterion(total, y)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

训练循环的每个迭代包含五个步骤：
1. 将数据移动到计算设备（CPU或GPU）
2. 前向传播计算预测值
3. 计算损失值
4. 反向传播计算梯度
5. 优化器更新参数

> **关键理解**：`optimizer.zero_grad()`必须在`loss.backward()`之前调用，否则梯度会累积而非替换。这是PyTorch初学者最容易犯的错误之一。

**步骤三：理解验证评估**

训练过程中定期调用`check_accuracy`方法评估模型在验证集上的表现：

```python
if verbose and t % print_every == 0:
    print('Iteration %d, loss = %.4f' % (t, loss.item()))
    self.check_accuracy(loader_val, model)
```

- `check_accuracy`方法将模型切换到评估模式（`model.eval()`）
- 使用`torch.no_grad()`禁用梯度计算，节省内存和加速推理
- 计算并打印验证集上的准确率

### 7.2 背景知识

超参数调优是深度学习模型开发中不可或缺的环节。与模型参数（通过训练自动学习）不同，超参数需要在训练前手动设定，直接影响模型的训练效率和最终性能。DeepFM的关键超参数包括嵌入维度、DNN隐藏层结构、Dropout率、学习率和训练轮数等。

### 8.2 背景知识

DeepFM模型训练完成后，每个类别特征值都会获得一个低维嵌入向量，这些向量编码了特征值的语义信息。在CTR预估场景中，如果两个广告位的嵌入向量非常接近，说明它们对用户点击行为的影响模式相似；如果某个特征值的嵌入向量远离其他值，说明该特征值具有独特的用户行为模式。

## 九、DeepFM与其他CTR模型的对比分析

### 9.1 实验目标

理解CTR预估模型的演化脉络，掌握DeepFM相对于Wide&Deep、FNN、PNN等模型的设计优势，能够设计对比实验验证DeepFM的优越性。

### 9.2 背景知识

CTR预估模型的演化经历了从线性模型到因子分解机，再到深度学习模型的几个重要阶段。早期的逻辑回归（LR）模型虽然简单高效，但无法自动学习特征交互，严重依赖人工特征工程。FM通过隐向量内积自动建模二阶特征交叉，大大减少了对人工特征工程的依赖，但受限于二阶交互。随着深度学习的发展，研究者提出了多种融合FM和DNN的模型。

Wide&Deep（2016年Google提出）是第一个将线性模型与DNN结合的CTR模型，Wide部分（线性模型）负责记忆，Deep部分（DNN）负责泛化。但Wide&Deep的Wide部分仍需要手工设计交叉特征。FNN（Factorization-machine supported Neural Network）先用FM预训练初始化DNN，再微调DNN，但FM和DNN的训练是分离的，无法端到端联合优化。PNN（Product-based Neural Network）在嵌入层和DNN之间引入乘积层，显式建模特征交叉，但乘积层的计算开销较大且灵活性不足。

DeepFM的核心创新在于：用FM替代Wide&Deep中的Wide部分，实现了二阶交互的自动学习；FM和DNN共享嵌入层，实现了端到端的联合训练；无需手工特征工程即可达到最优效果。这些设计使得DeepFM在多个基准数据集上均取得了优于前述模型的性能。

### 10.2 背景知识

原始DeepFM的PyTorch实现是一个教学级别的代码，存在多处可以改进的空间。首先，DNN隐藏层之间缺少激活函数（如ReLU），严重限制了模型的非线性表达能力。其次，评估指标仅使用准确率，在CTR预估场景下不够全面，应增加AUC和Log Loss。第三，训练过程没有实现早停（Early Stopping）机制，可能导致过拟合或浪费训练时间。此外，还可以引入更先进的技术，如注意力机制（让模型自动学习特征交互的重要性权重）、残差连接（缓解深层网络的梯度消失）等。

在实际工程中，CTR预估模型还需要考虑线上服务性能。DeepFM的推理延迟主要来自嵌入查找和DNN前向传播，可以通过模型量化、知识蒸馏等技术压缩模型大小、加速推理。

- 注意：最后一层（输出层）不应使用ReLU，因为Logit值可以为负

**步骤二：实现AUC评估指标**

在`check_accuracy`方法中添加AUC计算：

```python
from sklearn.metrics import roc_auc_score

all_preds = []
all_labels = []
model.eval()
with torch.no_grad():
    for xi, xv, y in loader:
        xi = xi.to(device=self.device, dtype=self.dtype)
        xv = xv.to(device=self.device, dtype=torch.float)
        total = model(xi, xv)
        probs = torch.sigmoid(total).cpu().numpy()
        all_preds.extend(probs.flatten())
        all_labels.extend(y.numpy().flatten())

auc = roc_auc_score(all_labels, all_preds)
print(f'AUC: {auc:.4f}')
```

**步骤三：实现早停机制**

在`fit`方法中增加早停逻辑：

```python
best_val_loss = float('inf')
patience = 3
patience_counter = 0

for epoch in range(epochs):
    # ... 训练逻辑 ...
    val_loss = evaluate_validation_loss()
    if val_loss < best_val_loss:
        best_val_loss = val_loss
        patience_counter = 0
        # 保存最佳模型
        torch.save(model.state_dict(), 'best_model.pth')
    else:
        patience_counter += 1
        if patience_counter >= patience:
            print(f'Early stopping at epoch {epoch}')
            break
```
