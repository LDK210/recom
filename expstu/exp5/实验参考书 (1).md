## 一、Estimator 与 Transformer 核心抽象

### 实验目标

深入理解 MLlib 中 **Estimator**（具有 `fit()` 方法，通过训练数据生成模型）与 **Transformer**（具有 `transform()` 方法，将一个 DataFrame 转换为另一个 DataFrame）的类层次结构；掌握通过 `ParamMap` 显式配置参数，通过 `explainParams()` 查看参数说明。

### 操作步骤

#### 1.1 创建 LogisticRegression Estimator

1. 从 `pyspark.ml.classification` 导入 `LogisticRegression`
2. 实例化一个 `LogisticRegression` 对象，指定 `featuresCol` 为特征列名（此处为 `"features"`，稍后将向量化的列）、`labelCol` 为 `"label"`
3. 调用该对象的 `explainParams()` 方法，观察所有可用参数及其默认值

> **提示**：此时无需真实数据，只需创建对象并查看参数说明。

#### 1.2 使用 ParamMap 配置参数

1. 从 `pyspark.ml.param` 导入 `ParamMap`
2. 创建一个 `ParamMap` 对象
3. 使用 `lr.maxIter` 和 `lr.regParam` 的方式，将 `maxIter` 设为 `100`，`regParam` 设为 `0.01`，添加到 ParamMap 中
4. 打印 ParamMap 的内容，确认参数已被正确设置

#### 1.3 直接设置参数与对比

1. 通过 `lr.setMaxIter(50).setRegParam(0.05)` 直接在 Estimator 对象上设置参数
2. 再次调用 `explainParams()`，观察参数值的变化
3. 对比"直接设置"与"ParamMap"两种方式的区别

### 关键 API

```python
# Estimator 创建
LogisticRegression(featuresCol="features", labelCol="label")

# 查看全部参数及说明
lr.explainParams()

# ParamMap 方式
ParamMap().put(lr.maxIter, 100).put(lr.regParam, 0.01)

# 直接设置方式
lr.setMaxIter(50).setRegParam(0.05)
```

> **思考题**：ParamMap 和直接设置参数两种方式各有什么优缺点？什么场景下更适合使用 ParamMap？

## 二、特征工程——文本分词与哈希特征转换

### 实验目标

理解 `Tokenizer` 将文本分割为单词序列的原理；理解 `HashingTF` 将词项映射为固定长度特征向量的机制，掌握词频特征提取在文本分类任务中的应用方式。

### 操作步骤

#### 2.1 加载文本数据

1. 使用 `spam_messages.csv` 加载数据，该文件包含 `label`（0 为正常短信、1 为垃圾短信）和 `message` 两列
2. 查看数据前几行，了解文本内容格式

#### 2.2 文本分词（Tokenizer）

1. 从 `pyspark.ml.feature` 导入 `Tokenizer`
2. 创建 `Tokenizer` 对象，设置 `inputCol` 为 `"message"`、`outputCol` 为 `"words"`
3. 对 DataFrame 调用 `transform()` 方法，查看分词结果
4. 观察分词后的 `words` 列，确认文本已被正确切分为单词数组

#### 2.3 词频哈希特征（HashingTF）

1. 从 `pyspark.ml.feature` 导入 `HashingTF`
2. 创建 `HashingTF` 对象，设置 `inputCol` 为 `"words"`（分词输出列）、`outputCol` 为 `"rawFeatures"`
3. 指定 `numFeatures` 参数为 `1000`（哈希桶数量）
4. 对上一步的输出 DataFrame 再次调用 `transform()` 方法
5. 观察生成的 `rawFeatures` 列，它是一个稀疏向量，代表每个样本的词频特征

### 关键 API

```python
# 文本分词
Tokenizer(inputCol="message", outputCol="words")
tokenizer.transform(df)

# 词频哈希特征提取
HashingTF(inputCol="words", outputCol="rawFeatures", numFeatures=1000)
hashingTF.transform(tokenized_df)
```

> **思考题**：`HashingTF` 的 `numFeatures` 设置过小或过大会带来什么问题？`HashingTF` 相比于 `CountVectorizer` 和 `TF-IDF` 有何优劣？

## 三、逻辑回归模型训练与预测

### 实验目标

掌握创建逻辑回归 Estimator 对象并设置相关训练参数；利用训练数据生成逻辑回归 Transformer 对象，预测输入数据；掌握 `fit()` 方法训练模型与 `transform()` 方法进行预测的基本流程。

### 操作步骤

#### 3.1 数据准备与向量化

1. 加载 `customer_loan.csv` 数据
2. 从 `pyspark.ml.feature` 导入 `VectorAssembler`
3. 将 `age`、`income`、`credit_score`、`loan_amount`、`employment_years`、`debt_ratio` 六列合并为 `"features"` 向量列
4. 使用 `VectorAssembler` 的 `transform()` 方法生成新的 DataFrame
5. 选择 `label` 和 `features` 列，确认数据格式正确

#### 3.2 训练逻辑回归模型

1. 创建 `LogisticRegression` 对象，设置 `maxIter=100`、`regParam=0.01`、`elasticNetParam=0.8`
2. 调用 `fit()` 方法，传入准备好的训练 DataFrame
3. `fit()` 的返回值是一个 `LogisticRegressionModel`（它是 Transformer 的子类）

#### 3.3 使用模型进行预测

1. 对训练好的模型调用 `transform()` 方法，传入训练数据
2. 查看预测结果，重点关注以下列：
   - `rawPrediction`：原始预测得分
   - `probability`：每个类别的概率
   - `prediction`：最终预测类别
3. 选择 `label`、`prediction`、`probability` 列，观察预测是否合理

### 关键 API

```python
# 特征向量组装
VectorAssembler(inputCols=["age","income","credit_score","loan_amount","employment_years","debt_ratio"],
                outputCol="features")

# 模型训练
lr = LogisticRegression(maxIter=100, regParam=0.01, elasticNetParam=0.8)
model = lr.fit(training_data)

# 模型预测
predictions = model.transform(data)
predictions.select("label", "prediction", "probability").show(10)
```

> **思考题**：`elasticNetParam` 取 0 时对应 L2 正则化（Ridge），取 1 时对应 L1 正则化（Lasso），取中间值时是什么正则化方式？尝试调整参数观察预测结果的变化。

## 四、模型训练摘要信息提取

### 实验目标

能够从逻辑回归模型的训练摘要（`summary` 对象）中提取迭代次数、系数矩阵、截距向量等关键信息；理解这些指标对于诊断模型收敛状态与特征重要性的参考价值。

### 操作步骤

#### 4.1 获取训练摘要

1. 使用第三节训练好的模型，访问其 `summary` 属性
2. 注意：`summary` 只能在训练数据上使用（通过 `fit()` 自动生成）

#### 4.2 提取关键信息

1. 查看模型的迭代次数（`summary.totalIterations`）
2. 查看目标函数的历史值（`summary.objectiveHistory`），观察损失是否逐渐下降
3. 提取模型的系数矩阵（`model.coefficientMatrix`）和截距向量（`model.interceptVector`）
4. 将系数与特征名称对应，分析哪些特征对预测影响最大（系数绝对值越大，影响越强）

#### 4.3 准确率与混淆矩阵

1. 从 summary 中获取准确率（`summary.accuracy`）
2. 查看按真实标签分组的统计信息（`summary.predictionsByLabel`）
3. 查看真阳性率、假阳性率等指标（`summary.truePositiveRateByLabel`、`summary.falsePositiveRateByLabel`）
4. 打印接收者操作特征（ROC）曲线下的面积（`summary.areaUnderROC`），评估模型整体区分能力

### 关键 API

```python
# 训练摘要
summary = model.summary

# 迭代与收敛信息
summary.totalIterations
summary.objectiveHistory       # List[float]，目标函数值序列

# 模型参数
model.coefficientMatrix        # 系数矩阵（稀疏矩阵）
model.interceptVector          # 截距向量

# 评估指标
summary.accuracy
summary.areaUnderROC
summary.truePositiveRateByLabel
summary.falsePositiveRateByLabel
```

> **思考题**：如果 `objectiveHistory` 中的损失值没有明显下降，可能是什么原因？你会如何调整参数？

## 五、ML 管道技术——文本逻辑回归分类器

### 实验目标

理解 `Pipeline` 与 `PipelineModel` 的工作机制，能够将 `Tokenizer`、`HashingTF` 等特征转换器与 `LogisticRegression` 等估计器按序组装为完整工作流；掌握管道模型对测试数据的端到端预测流程。

### 操作步骤

#### 5.1 加载与划分数据

1. 加载 `spam_messages.csv`，对 `label` 列进行类型转换（确保为 Double 类型）
2. 使用 `randomSplit()` 按 8:2 的比例划分为训练集和测试集，设置随机种子 `seed=42`
3. 分别打印训练集和测试集的行数

#### 5.2 组装 Pipeline

1. 从 `pyspark.ml` 导入 `Pipeline`
2. 创建三个 Stage：
   - **Stage 1**：`Tokenizer`，将 `message` 列分词为 `words` 列
   - **Stage 2**：`HashingTF`，将 `words` 列转换为 `features` 列（`numFeatures=1000`）
   - **Stage 3**：`LogisticRegression`，设置 `maxIter=100`、`regParam=0.01`
3. 使用 `Pipeline` 将这三个 Stage 按顺序组装
4. 调用 Pipeline 的 `fit()` 方法，传入训练集，获得 `PipelineModel`

#### 5.3 使用管道模型预测

1. 对 PipelineModel 调用 `transform()` 方法，传入测试集
2. 查看预测结果，选择 `label`、`message`、`prediction`、`probability` 列展示
3. 对比真实标签与预测标签，观察分类效果
4. 计算并打印测试集上的准确率（可以手动计算或使用 `evaluator`）

### 关键 API

```python
# 数据划分
train_data, test_data = df.randomSplit([0.8, 0.2], seed=42)

# Pipeline 组装
Pipeline(stages=[tokenizer, hashingTF, lr])
pipeline_model = pipeline.fit(train_data)

# 端到端预测
predictions = pipeline_model.transform(test_data)
```

> **思考题**：Pipeline 中的 Stage 顺序是否可以随意调整？如果将 LogisticRegression 放在 Tokenizer 之前会发生什么？为什么？

## 六、模型持久化——保存与加载

### 实验目标

学会使用 `write().save()` 持久化模型到本地文件系统，以及通过 `read().load()` 重新加载模型，理解模型持久化在工程实践中的必要性。

### 操作步骤

#### 6.1 保存 PipelineModel

1. 使用第五节训练得到的 PipelineModel
2. 调用 `model.write().overwrite().save("spam_pipeline_model")` 将模型保存到指定目录
3. 使用 `ls` 或文件管理器查看生成的目录结构，理解模型持久化的存储方式

#### 6.2 加载 PipelineModel

1. 从 `pyspark.ml` 导入 `PipelineModel`
2. 使用 `PipelineModel.read().load("spam_pipeline_model")` 加载已保存的模型
3. 对加载的模型调用 `transform()` 方法，传入测试集
4. 将加载模型后的预测结果与第五节原始模型的预测结果进行对比，验证一致性

#### 6.3 保存与加载单个 LogisticRegressionModel

1. 单独保存第三节训练的 LogisticRegressionModel：`model.write().overwrite().save("lr_model")`
2. 从 `pyspark.ml.classification` 导入 `LogisticRegressionModel`
3. 使用 `LogisticRegressionModel.read().load("lr_model")` 加载模型
4. 加载后查看模型的系数和截距，与原始模型对比

### 关键 API

```python
# 保存模型（overwrite 覆盖已有模型）
pipeline_model.write().overwrite().save("spam_pipeline_model")

# 加载 PipelineModel
PipelineModel.read().load("spam_pipeline_model")

# 保存单个模型
lr_model.write().overwrite().save("lr_model")

# 加载单个模型
LogisticRegressionModel.read().load("lr_model")
```

> **思考题**：为什么要使用 `overwrite()` 选项？在实际项目中，如果不加 overwrite 直接保存一个同名模型会发生什么？

## 七、统计计算——均值、范数与相关矩阵

### 实验目标

利用 `ml.stat` 模块计算均值、L1 范数、L2 范数、皮尔逊相关矩阵、斯皮尔曼相关系数；掌握 `Summarizer` 工具对特征向量的多维度统计描述能力；理解不同相关系数的计算差异及适用场景。

### 操作步骤

#### 7.1 数据准备

1. 加载 `health_data.csv`，该文件包含身高、体重、血压、胆固醇、血糖、BMI 六列数值型特征
2. 使用 `VectorAssembler` 将所有列合并为 `"features"` 向量列

#### 7.2 Summarizer 统计

1. 从 `pyspark.ml.stat` 导入 `Summarizer`
2. 对特征列分别计算：
   - **均值**（`mean()`）：每个特征的平均值
   - **方差**（`variance()`）：每个特征的方差
   - **L1 范数**（`norm("l1")`）：每个样本特征向量的一范数（绝对值之和）
   - **L2 范数**（`norm("l2")`）：每个样本特征向量的二范数（欧氏长度）
   - **最大值**（`max()`）和 **最小值**（`min()`）
3. 将统计结果打印输出，分析各特征的分布情况

#### 7.3 皮尔逊相关矩阵

1. 从 `pyspark.ml.stat` 导入 `Correlation`
2. 使用 `Correlation.corr(data, "features", method="pearson")` 计算皮尔逊相关系数矩阵
3. 将结果从 DenseMatrix 转换为可读格式，打印相关矩阵
4. 分析哪些特征之间存在强正相关或强负相关

#### 7.4 斯皮尔曼相关系数

1. 使用 `Correlation.corr(data, "features", method="spearman")` 计算斯皮尔曼相关系数
2. 与皮尔逊结果对比，观察两种相关系数的差异
3. 思考：哪些情况下斯皮尔曼相关系数比皮尔逊更合适？

### 关键 API

```python
# Summarizer 多维度统计
from pyspark.ml.stat import Summarizer

df.select(
    Summarizer.summarizer(df["features"], "mean", "variance", "norm(l1)", "norm(l2)", "max", "min")
).show(truncate=False)

# 皮尔逊相关矩阵
Correlation.corr(data, "features", method="pearson")

# 斯皮尔曼相关矩阵
Correlation.corr(data, "features", method="spearman")

# DenseMatrix 转换为数组
corr_matrix.values.reshape(n, n)
```

> **思考题**：皮尔逊相关系数度量的是线性关系，而斯皮尔曼度量的是单调关系。如果两个变量之间呈 U 型关系，这两种相关系数会分别给出什么结果？

## 八、RDD 分层抽样与随机数生成

### 实验目标

实现 RDD 的分层抽样与随机数生成；掌握 `sampleByKey()` 方法按键进行分层抽样的原理与参数配置；理解 `RandomRDDs` 生成服从特定分布随机数的方法，认识 RDD 基础操作在数据预处理阶段的独立价值。

### 操作步骤

#### 8.1 准备 RDD 数据

1. 加载 `sales_data.csv` 为 DataFrame
2. 按 `category` 列进行分组计数，查看各类别的数据量分布
3. 将 DataFrame 转换为 RDD（使用 `rdd` 属性），或直接使用 DataFrame 进行后续操作

#### 8.2 分层抽样（sampleByKey）

1. 从 `pyspark.sql.functions` 导入 `col`，定义分层抽样比例：
   - `fractions = {"电子产品": 0.5, "服装": 0.3, "食品": 0.4}`
2. 将 DataFrame 转换为 RDD，并映射为 `(key, row)` 格式的 PairRDD
3. 使用 `rdd.sampleByKey(False, fractions, seed=42)` 进行分层抽样
4. 统计抽样后各类别的数量，与预期比例进行对比验证
5. 将抽样结果转回 DataFrame 并 `show()`

#### 8.3 随机数生成（RandomRDDs）

1. 从 `pyspark.mllib.random` 导入 `RandomRDDs`
2. 获取 `SparkContext`（通过 `spark.sparkContext`）
3. 生成服从标准正态分布的随机数 RDD：`RandomRDDs.normalRDD(sc, 1000, seed=42)`
4. 对随机数 RDD 进行基本统计：计算均值和标准差
5. 生成服从均匀分布 U(0, 1) 的随机数 RDD：`RandomRDDs.uniformRDD(sc, 1000, seed=42)`
6. 验证均匀分布随机数的范围是否在 [0, 1) 之间

#### 8.4 ML API 与 MLlib API 的对比使用

1. 在同一个 Spark 应用中，同时展示：
   - **ML API（DataFrame-based）**：使用 Pipeline 进行文本分类（第五节内容回顾）
   - **MLlib API（RDD-based）**：使用 RandomRDDs 生成随机数、使用 sampleByKey 进行抽样
2. 观察 `spark`（SparkSession）与 `sc`（SparkContext）的关系：`spark.sparkContext`
3. 总结两套 API 各自适用的场景
